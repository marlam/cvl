October 16, 2007

CVL and Cvtool 0.2.2 are released!
This release mainly fixes some bugs, but it also adds minor new features. See the ChangeLog for details.


October 3, 2007

CVLView 0.1.2 is released!
This release fixes some bugs. See the ChangeLog file for details.


September 9, 2007

Cvtool 0.2.1 is released, together with CVLView 0.1.1 and CVLTonemap 0.1.1!
These releases fix some bugs, including build failures, potential segmentation faults, and wrong computation results for some input images. See the ChangeLog files for details.
Additionally, CVLTonemap now handles postprocessing parameters for each tone mapping method separately, and allows to save and load parameter sets.


August 31, 2007

Cvtool 0.2.0 is released, together with CVLView 0.1.0 and CVLTonemap 0.1.0!
This is a complete rewrite. All computations are now done on the graphics processing unit (GPU). Additionally, all programs can now handle floating point data, e.g. for high dynamic range imaging.


November 6, 2006

Cvtool 0.1.0 is released!
Some bug fixes, performance improvements, and functionality enhancements were made.
Please see the file NEWS for a more complete overview of the changes.


July 18, 2006

Cvtool 0.0.3 is released!
Among the new features are 2D and 3D skeletonization and medial axis transforms.
Many cleanups were made and many bugs were fixed.
Please see the file NEWS for a more complete overview of the changes.


July 5, 2006

Cvtool development has switched from CVS to Subversion. See the public Subversion repository for cvtool on SourceForge.


May 27, 2006

Cvtool 0.0.2 is released!
A lot of new functionality was added, for example for layering, drawing graphics and text, and Squared Euclidean Distance Transforms (SEDT).
Many cleanups were made and many bugs were fixed.
Please see the file NEWS for a more complete overview of the changes.


Jan 19, 2006

Cvtool 0.0.1 is released!
This is the first public release. Cvtool is still in an early alpha phase, but a lot of useful commands are already implemented and fully documented. See the documentation for details and examples.
